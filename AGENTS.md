# Charlie's AI Protocols

AI multi-agent collaboration system with specialized agents, powered by CrewAI.

## Core Protocol

See `docs/agent-skills/00-core-protocol.md` for the global constitution that all agents must follow.

## Git Workflow

See `docs/policies/git-workflow.md` for Conventional Commits, branching strategy, and PR conventions.

## Project Structure

- `docs/agent-skills/*-agent.md` — Agent system prompts (SSOT), picked up by `factory.py`.
- `docs/agent-skills/00-core-protocol.md` — Global constitution (NOT an agent), injected as shared preamble.
- `docs/agent-skills/strategies/` — Framework-specific tactics (NOT agents).
- `docs/policies/` — Cross-tool policies, readable by any AI CLI.
- `engine/` — Python 3.10+ CrewAI >= 1.14 execution engine.
- `workspace/` — Gitignored agent output sandbox.

### Invoking a skill

Use `$skill_name` to load the agent's full system prompt on demand:

```
$qa 請幫我針對這段 API 寫單元測試。
```

### Chaining skills

Combine multiple skills in a single prompt for cross-role workflows:

```
請使用 $security 檢查目前的登入模組。
檢查完成後，請切換為 $logger 的角色，將資安稽核結果寫入 workspace/history/audit-log.md。
```

UI 設計任務若需要可維護設計資產，應要求 `03-ui-agent` 同步輸出：

```
請使用 $ui 為 auth 模組產出 UI Spec、tokens.json、screens.json 與 prototype.html，供 Figma / Claude Design / Frontend 後續接手。
```

若要啟用第二層同步到 Figma，可在設計資產完成後交由 `$figma` 處理：

```
請先使用 $ui 為 auth 模組產出 UI Spec、tokens.json、screens.json 與 prototype.html。
完成後切換為 $figma，使用 MCP 或 import script 同步到指定的 Figma page。
```

The agent definitions are loaded progressively — only the invoked skills consume tokens.

## Conventions

- All communication with the user must be in **Traditional Chinese (繁體中文)**.
- Agent files must use `*-agent.md` naming to be instantiated by CrewAI.
- Short aliases (`qa.md`, `security.md`, ...) are generated by `scripts/mapper.sh` for `$` invocation.
- Commit messages follow Conventional Commits: `<type>(<scope>): <subject>`.
